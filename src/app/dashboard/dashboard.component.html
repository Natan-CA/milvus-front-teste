Introdução

Bem-vindo ao teste técnico do processo seletivo Milvus!
Neste teste, será avaliado sua competência técnica em Angular+, onde será considerado tanto seu conhecimento do framework e seus componentes, como sua organização e lógica.
O teste não é longo, mas é bem detalhado. Preste atenção nas regras, mas faça com calma! O que será avaliado é sua experiência e comportamento com a tecnologia, e não a conclusão do teste.
Faça com a sua personalidade e visão organizacional, tanto das telas quanto dos arquivos, queremos saber como você pensaria e se organizaria em uma tarefa do nosso cotidiano.

Requisitos

Você terá 2 (duas) horas para realizar o seguinte cronograma:
  • Criar 4 (quatro) componentes (Use o Angular CLI para criar mais rapidamente) dentro da pasta app/user (nesta pasta já há um submódulo importado no módulo principal da aplicação, importe os componentes neste submódulo que eles já serão importados no módulo principal), sendo eles:
  • Uma tela de listagem de usuários:
    Nesta tela, você apresentará uma tabela com a lista de usuários cadastrados no sistema, com seus principais atributos (CPF, Nome, E-mail, Status, e mais duas colunas à sua escolha) e uma última coluna com um botão de opções, faça um dropdown com a opção de Editar e Excluir, além de uma linha abaixo para os filtros, veja na parte de API quais os filtros habilitados. Acima da tabela, apresente um botão de Adicionar, para cadastrar um novo usuário.
  • Uma tela de detalhes do usuário:
    Esta tela será redirecionada à partir do clique do usuário na linha da tabela referente ao usuário a ser mostrado da listagem de usuários.
    Nesta tela você apresentará todos os dados do usuário, mas lembre-se que deverá ser apenas para amostragem dos dados. Organize da forma que achar melhor. Além disso, faça um botão para voltar para a tela de listagem, e um botão de opções de Edição e Exclusão. Posicione estes botões da forma que achar melhor (isso será avaliado).
  • Uma tela de edição do usuário:
    Nesta tela, apresente os mesmos dados apresentados na tela de detalhamento do usuário, mas com a possibilidade do mesmo editar qualquer campo que ele deseja. Faça um botão de Cancelar Edição, retornando o usuário para a tela de listagem, e um botão de Salvar.
  • Uma tela de cadastro do usuário:
    Nesta tela, apresente os mesmos campos necessários para o cadastro do usuário como no exemplo retornado pela API. Você pode organizar os campos da mesma forma da tela de edição, facilitando seu desenvolvimento. Apresente novamente um botão de Cancelar Cadastro, retornando o usuário para a tela de listagem, e um botão de Salvar.
  • Crie um novo link dentro da sidenav, abaixo do item API e Exemplos, que redirecione o usuário para a listagem.
  • Use o arquivo app/app-routing.module.ts para criar as rotas da sua aplicação.
  • Crie dentro da pasta app/services, uma Service com todas as requisições relacionados ao usuário para a API, e chame em cada um dos componentes conforme necessidade.
  • Confira na página de API e Exemplos, os endereços da API que você irá usar, e também alguns exemplos para referência.
  • A feitura de cada um dos itens exatamente como especificado conta positivamente para a avaliação!
Neste projeto, já vem instalado o Angular Material, use ele para facilitar seu desenvolvimento, como por exemplo suas tabelas, formulários, botões, ícones, entre outros componentes que deseja usar. A importação do Angualar Material é feita através do arquivo app/shared/material.module.ts, ele já possui alguns componentes importados e já é instanciado como CustomMaterialModule, nos arquivos app/app.module.ts e app/user/user.module.ts. Se desejar usar outro componente que não esteja importado neste arquivo, vá até o Site Oficial e veja como usar o componente que deseja (ele também pode ser usado como guia para uso dos componentes já importados).
Também está instalado no projeto o Bootstrap 4, use como quiser.
Além disso, o projeto também inclui o NGX Mask e NGX Toastr. Se houver tempo, use máscaras nos campos do usuário que necessitem delas, como por exemplo o campo CNPJ. E renderize notificações de sucesso ou erro nas ações de salvar e excluir para o usuário. Os links podem ser usados como guia de uso das ferramentas.
O Typescript deste projeto segue as regras do ES2017, e o estilo é feito por meio do SASS.
A organização do Typescript, bem como do estilo empregados no código será de suma importância na avaliação, fique atento!
Não esqueça de disponibilizar seu código para nós! Suba o projeto por qualquer repositório de sua preferência e nos envie o link quando finalizar o teste, iremos avaliá-lo.
Boa sorte!
